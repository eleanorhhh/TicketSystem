# 搶票系統
### 專案動機與適法性
為了解決熱門票券搶購困難的問題，我自主開發了一套自動化搶票系統。考量到相關法規限制，本專案全程於合法的「搶票練習網站」進行測試與開發。

### 效能瓶頸與優化策略

- 跳過 DOM 節點搜尋： 初期採用爬蟲尋找網頁按鈕並模擬點擊，但實測後發現 DOM 渲染與搜尋的延遲會大幅降低搶票成功率。經由開發者工具（DevTools）分析後，發現核心按鈕僅是執行網址重導向（Redirect）。因此，我將策略改為「直接對目標選位網址發出請求」，省去前端 UI 的加載與尋找時間。

- 預載組態參數： 為了進一步縮短流程，我增加了訂位資訊預填機制。讓使用者在腳本執行前先設定好目標票數與座位區域，當進入選位頁面時系統能直接帶入參數，顯著提升了整體的執行速度。
